## 管理后台登录失效自动跳转登录页（admin/index.html + src/index.ts）

### 背景
用户反馈：检测到登录失败应直接跳转到登录页面。

### 分析
- 后端 `src/middleware/auth.ts` 认证失败统一返回 `code: -2`，HTTP 状态码可能是 401（未授权/token 失效）或 403（权限不足，如 adminMiddleware/merchantMiddleware）。
- 前端 `api.request` 原先只检查 `response.status === 401`，未覆盖 HTTP 200/403 但业务码 `code === -2` 的情况。
- `admin/index.html` 与 `src/index.ts` 内嵌的 HTML/JS 是两份需同步维护的副本。

### 改动
在两份文件的 `api.request` 通用请求方法中增强登录失效检测：
1. HTTP 401 → 清除认证 + `navigateTo('login')`（保持原逻辑）。
2. HTTP 403 或 200 时，克隆响应读取 body，若 `data.code === -2` → 清除认证 + 跳转登录页。
3. 用 `response.clone()` 读取，避免消费原始 body 影响调用方。
4. 非 JSON 响应静默忽略，交由调用方处理。

### 涉及位置
- `admin/index.html` 约 1432-1466 行
- `src/index.ts` 约 1662-1696 行（内嵌副本，同步修改）

## 登录页面居中修复（admin/index.html + src/index.ts）

### 问题
登录页面未能在视口中居中显示。

### 原因
`.layout` 容器使用了 `display: flex; min-height: 100vh;`，登录页面的外层 div 作为其 flex 子元素，未设置 `width: 100%` 导致无法正确占满容器宽度，居中效果失效。

### 修复
在 `renderLogin()` 函数返回的外层 div style 中添加 `width: 100%`：
- `admin/index.html` 约 1362 行
- `src/index.ts` 约 1592 行

## 后台首页空白修复（src/index.ts）

### 问题
用户反馈：后台首页一片空白，什么都没有。

### 根因
`src/index.ts` 的 `/admin` 路由用 TS 模板字符串（反引号）内联了整个后台 HTML+JS。内联 JS 字符串拼接中，HTML `onclick` 属性里调用 `navigateTo('orders')` 等函数时，参数单引号用 `\'` 转义（如 `navigateTo(\'orders\')`）。但在 TS 模板字符串里，`\'` 的反斜杠会被吃掉（`\'` → `'`），导致浏览器实际收到 `navigateTo('orders')`，单引号提前终止了外层 JS 字符串，引发 **`SyntaxError: Unexpected identifier 'orders'`**。整个 `<script>` 块不执行 → `DOMContentLoaded` 不触发 → `navigateTo` 不执行 → `mainContent` 保持初始空白。

### 诊断方法
- `node --check` 提取的脚本通过（因提取的是源文件原始文本，`\'` 作为 JS 合法）。
- 用 jsdom 解析实际 HTML 输出 + `vm.Script` 编译，才暴露语法错误及精确行号（740 行 `navigateTo('orders')`）。
- 关键：TS 模板字符串转义与纯 JS 解析存在差异，`node --check` 会漏报。

### 修复
把内联 script（1553~4203 行）中所有 `\'`（单反斜杠+单引号）改为 `\\'`（双反斜杠+单引号）。共 16 行、56 处。模板字符串输出 `\'`，浏览器 JS 正确解析为转义单引号。
- 涉及 `renderDashboard/renderMerchants/renderOrders/renderSettlements/renderPaymentTypes/renderChannels/renderLogs` 中 `onclick` 内带字符串参数的调用。

### 验证
jsdom 执行修复后页面：未登录 → 正确渲染登录页；登录态 → dashboard 骨架渲染（mainContent 62→4004 字节）+ 数据加载（todayOrders 显示 5）。

### 注意
- `admin/index.html` 是另一份独立副本（用模板字符串/async），不受此问题影响，但两份需同步维护。
- `dist/index.js` 为 tsc 旧产物，wrangler 用 `src/index.ts`，不影响运行；如需更新可 `npm run build`。
